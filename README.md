# memorylog
## **プロジェクト概要**

**プロジェクト名：**

MemoryLog(めもろぐ)

**概要説明：**

日常で出会った人や、一緒に過ごしたイベントを記録することで、

「誰とどんな時間を過ごしたか」を思い出せる個人向け記録アプリ。

**開発目的：**

- 人との会話や出会いを記録して忘れないようにする
- 人間関係を思い出しやすくする
- イベントや思い出を整理する
- 人とのつながりを振り返ることができる

**開発期間：**

2026年3月〜(個人開発)

**開発体制：**

- フロントエンド：自分
- バックエンド：自分
- デザイン：自分

---

## **1. 目的・ゴール定義**

**1.1 プロダクトの目的**

「誰とどんな時間を過ごしたか」を簡単に思い出せるアプリを提供する。

---

## **2. ターゲットユーザー**

**2.1 ペルソナ**

| 属性 | 内容 |
| --- | --- |
| 名前 | さくら |
| 年齢 | 20歳 |
| 職業 | 大学生 |
| 使用シーン | 友達と遊んだあと |
| 課題／ニーズ | 以前会った人との会話内容や出会った場所を忘れてしまう |
| 使用動機 | 人との思い出を整理して記録したい |

**2.2 ユーザーストーリー**

「私は友達や知り合いとの出来事を忘れないようにするために、このアプリを使う。」

---

## **3. 課題と価値仮説**

**3.1 現状の課題**

- 出会った人の情報を忘れてしまう
- 誰といつ遊んだか思い出しにくい
- 写真だけでは思い出の整理ができない

**3.2 提供価値**

- 人とイベントを紐づけて記録できる
- 人のページから「一緒に過ごしたイベント」を確認できる
- イベントの写真やメモを残せる
- 検索機能で人やイベントをすぐ見つけられる

## **9. 懸念点と対策**

| 懸念点 | 対策 |
| --- | --- |
| **個人情報の管理** | ユーザーごとにデータを分離し、他ユーザーのデータにアクセスできない設計にする |
| **入力の手間** | 人登録やイベント作成をシンプルなUIにし、入力項目を最小限にする |
| **継続利用の動機付け** | 思い出タイムラインや「今年よく会った人」など振り返り機能を追加する |
| **機能の多さ** | MVP（Minimum Viable Product）に絞って開発を進める
人登録・イベント作成・参加者追加 |
| **検索性の低さ** | 人名・メモ・場所などから検索できる機能を実装する |
| **誰のためのアプリか曖昧** | 「人との思い出を記録する個人アプリ」というコンセプトを明確にする |
